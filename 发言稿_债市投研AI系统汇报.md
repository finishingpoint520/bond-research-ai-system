# 债市投研AI系统建设汇报发言稿

> 发言时长：约 10 分钟（约 2200 字）
> 汇报人：研究中心（固收）
> 对象：公司领导、投资总监

---

## 一、我们在做什么（1 分钟）

各位领导，今天汇报的内容，不是"我们用 AI 写了什么报告"，而是我们在过去一年里，尝试重新定义债券投研的底层工作方式。

传统投研流程大家都很熟悉——研究员收集数据、分析判断、撰写报告、交叉验证。一个完整的判断周期，大约需要一个人大半天的时间，覆盖的维度通常不超过三个。

我们做的事情，概括为一句话：**让 AI 不只加速计算，更参与判断**。具体来说，是并行推进两条工作流——一条是定性框架的 Agent 化，即"五因子多 Agent 策略审计系统"；一条是定量信号的体系化，即"AI 量化模型系统"。两条线共享同一套数据底座，目标是把研究员从"生产者"升级为"审核者"。

---

## 二、整体架构（1 分钟）

两条工作流的分工是清晰的。

**五因子多 Agent 策略审计系统**，解决的是一套定性分析框架如何被标准化执行的问题。我们把债券研究员日常判断市场时会看的五个维度——基本面、政策面、流动性、市场情绪、机构行为——分别交给五个独立的 AI Agent 去分析。每个 Agent 输出四维评分：当前状态、边际变化、预期差和置信度。然后由一个总控 Agent 汇总，识别因子之间的冲突，生成策略建议，最后交给一个专门的审计 Agent 去挑错——检查数据是否可靠、逻辑是否自洽、有没有被忽视的反面证据。

**AI 量化模型系统**，解决的是定量信号的体系化问题。它包含了宏观景气、流动性、机构情绪等方向的量化模型，输出的是可以每日追踪、可以回测验证的数值信号。

两条线的关系是：量化模型提供客观信号，Agent 审计系统把这些信号纳入完整的分析框架中整合判断，最终由人工复核形成策略决策。这个闭环，是我们认为 AI 在投研领域能够真正产生价值的核心逻辑。

## 三、八个已交付子模块（6 分钟）

下面逐一介绍目前已完成的八个子模块。

### 1. 五因子多 Agent 策略审计系统（核心框架）

这是整个定性分析体系的骨架。它的设计思路非常朴素：我们不信任任何一个单一模型给出的结论，所以我们让多个 Agent 互相校验。

这个系统包含 12 个 Agent：5 个因子 Agent 各自打分，5 个增强 Agent 分别负责市场状态识别、拥挤度评估、历史案例检索、估值赔率判断和主导叙事提取，再加上总控和审计。每个 Agent 只输出自己负责的结构化字段，不写长篇大论，从而避免互相污染。

**当前的困难**：系统设计已完成，但尚未跑通端到端的全流程。原因是 P0 指标的数据管道还在与 IT 同事对接中——这是典型的"数据瓶颈"问题，不是设计问题。

### 2. 政策-舆情-预期差 Skill（v3）

这个模块要解决的是一个问题：央行的政策表态、政治局会议的措辞、市场舆论的热度——这些非结构化信息——如何被量化成可比较的信号？

我们的做法是建立一个政策语义词典，把政策表述映射为方向性评分，然后比较实际政策信号与市场预期的差距，形成"预期差"指标。这个模块已经迭代到第三版，完成了两年的历史回测。与五因子系统的关系是：它直接为政策面因子 Agent 提供 surprise_score 的量化输入。

**当前的困难**：政策文本的语义边界有时不够清晰，比如"稳健偏松"和"灵活适度"之间的量化差异还需要更细的标注。

### 3. 宏观高频景气指数（v2）

这是一个经典的宏观量化工具。追踪经济增长、通胀、融资需求等维度的日度和周度高频数据，合成一个综合景气判断。已迭代至第二版，为五因子系统中的基本面因子提供客观参考。

**与其他模块的关联**：它和流动性模型、情绪指数一起，构成了量化模型系统的"三大引擎"。三者的信号目前各自独立输出，后续会尝试融合为一个综合量化信号。

**当前的困难**：等权加权、AI 赋权、投票法都试过了，综合信号的回测效果不理想。原因在于三个底层模型的构造逻辑差异较大——景气指数是趋势型指标，情绪指数是择时型指标，流动性模型介于两者之间。强行融合反而稀释了各自的优势。等流动性模型完成新版本迭代后，会重新尝试。

### 4. 流动性量化模型

这个模型聚焦于资金利率——DR007、存单利率、OMO 操作——对债市的影响。目前正在从月度信号升级为日度信号，并增加了信号强度值。新版本还计划与存单利率模型关联，形成从"央行操作→资金价格→存单定价→债市影响"的完整传导链。

**与机构情绪指数的关系**：流动性是情绪的放大器。当资金面宽松但机构情绪偏冷时，市场往往处于等待状态；当两者共振时，方向最确定。这两个模型的交叉验证，是我们判断市场状态的核心依据。

**当前的困难**：新模型的 Y 值设定方案和预测周期还在与 IT 同事讨论中，候选指标测试需要时间。

### 5. 机构情绪择时指数（已上线）

这个指数的设计目标是：从机构行为和市场情绪指标中，找出对 10 年国债有择时意义的因子，对下一阶段的方向发出信号。它已经上线，每日通过邮件自动推送。

**与机构情绪同步指数的区别**：择时指数追求领先性，目标是"预测方向"；同步指数追求一致性，目标是"衡量热度"。两个体系互补——择时告诉你要不要动，同步告诉你现在的市场有多拥挤。

### 6. 机构情绪同步指数（拟合中）

前面提到了它和择时指数的分工。补充一点设计逻辑：为什么需要同步指数？因为很多指标本身没有很好的择时能力，但它们与债市走势高度相关。构建同步指数可以帮助衡量当前市场整体交易热度的位置——是过冷、中性还是过热。当同步指数显示过热时，不意味着马上反转，但意味着市场对利空会更加敏感，需要做仓位管理。

**当前的困难**：主导因子的提取和拐点识别策略还在探索中，目前只有一个初步拟合的版本。

### 7. 银行资负 Skill

这个模块专门分析银行资产负债行为对债市的影响。回答的问题是：当实体信贷需求偏弱时，银行是否通过增加债券投资来维持资产扩张？如果是，这究竟是利多债市——因为配置盘稳定——还是隐含利空——因为根子在于实体融资需求不足？

它需要在五因子框架下同时考虑机构行为和基本面两个维度，不能简单归到某一个因子下面。

**当前的困难**：框架初步形成，但数据链路和校验逻辑还需要进一步完善。

### 8. 跨资产周度复盘 Skill

最后一个模块，解决的是一个容易被忽视的问题：债券研究往往局限在债券市场内部，而忽略了股票、商品、汇率等资产类别传递的信号。这个 Skill 整合了债券、股票、商品、汇率的周度信号，形成跨市场验证闭环。

**设计思路**：单一资产的信号可能有噪声，但如果多个资产同时指向同一个方向，确定性会显著增强。反过来，如果跨资产信号出现冲突，也是重要的风险提示。

**当前的困难**：初步版本已形成，但跨资产的信号融合权重还需要积累更多样本后才能稳定。

## 四、当前的进展和下一步（1 分钟）

总结一下：八个子模块中，机构情绪择时指数已进入每日运行状态，政策 Skill 完成了两年回测，总体架构设计已经完成。目前的核心瓶颈不在方法论层面，而在数据管道和工程化落地层面——指标字典需要与 IT 协作标准化，部分模型需要完成新版本迭代后才能参与闭环运行。

接下来的六个月，我们的计划是：七月实现五因子系统的首次端到端运行；九月完成三个量化模型的综合信号融合和首次月度复盘；年底争取让策略信号进入实战状态，辅助久期和仓位的决策。

我们追求的不是一次性做一个漂亮的 Demo，而是建设一个可以持续运行、每周产出、每月迭代的生产系统。这条路没有捷径，但方向是对的。

谢谢各位。
